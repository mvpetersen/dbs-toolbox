# DBS Toolbox Examples

This directory contains example scripts demonstrating how to programmatically use the DBS Toolbox coordinate transformation utilities.

## Examples

### 1. Transform Surgical CSV: Stereotactic Space → T1W Space

Transforms surgical planning data from Leksell stereotactic frame coordinates to T1-weighted MRI space.

**Script:** [`transform_surgical_csv.py`](transform_surgical_csv.py)

**Usage:**
```bash
python transform_surgical_csv.py \
    data/surgical_data.csv \
    data/frame_registration.json \
    data/ct_to_t1w_0GenericAffine.mat \
    data/surgical_data_t1w.csv
```

**What it does:**
- Loads surgical planning data (target coordinates, ring/arc angles, MER tracks)
- Applies frame registration transform (frame → CT space)
- Applies ANTs transform (CT → T1W space)
- Outputs transformed coordinates with virtual entry points and MER track positions

**Input format (CSV):**
```csv
patient_id,hemisphere,target_structure,x,y,z,ring,arc,clinical_track,clinical_depth
PT001,left,STN,95.5,105.2,80.0,70.0,45.0,central,5.0
```

**Output includes:**
- Transformed target coordinates (x, y, z in T1W)
- Virtual entry points (entry_x, entry_y, entry_z)
- Original coordinates preserved (*_original)
- MER track positions if provided

### 2. Transform Electrode JSON: Postop CT → Preop CT → T1W Space

Transforms PyPaCER electrode reconstruction from postoperative CT space to T1-weighted MRI space through a two-step registration.

**Script:** [`transform_electrode_json.py`](transform_electrode_json.py)

**Usage:**
```bash
python transform_electrode_json.py \
    data/electrode_reconstruction.json \
    data/postop_to_preop_0GenericAffine.mat \
    data/preop_to_t1w_0GenericAffine.mat \
    data/electrode_reconstruction_t1w.json
```

**What it does:**
- Loads PyPaCER electrode reconstruction data (in postop CT space)
- Applies first ANTs transform (postop CT → preop CT)
- Applies second ANTs transform (preop CT → T1W)
- Transforms all electrode components: contacts, tips, entries, trajectories
- Outputs transformed reconstruction with original coordinates preserved

**Why two transforms?**
In typical DBS imaging workflows:
1. Electrodes are detected in postoperative CT
2. Postop CT is registered to preoperative CT (better quality, less metal artifact)
3. Preop CT is registered to T1W MRI (anatomical reference)
This two-step process provides better registration quality than direct postop→T1W

**Input format (JSON):**
```json
{
  "electrodes": [
    {
      "side": "left",
      "contact_positions_3d": [[x, y, z], ...],
      "tip_position": [x, y, z],
      "entry_position": [x, y, z],
      ...
    }
  ]
}
```

**Output includes:**
- All coordinates transformed to T1W space
- Original coordinates preserved (*_original fields)
- Updated metadata with transformation info

## Transform Files

You'll need transformation files from your own data:

### Frame Registration (`.json`)
Generated by Leksell Frame Registration tool. Contains the transformation matrix from stereotactic frame space to CT space.

### ANTs Transform (`.mat` or `.nii.gz`)
Generated by ANTs registration (e.g., `antsRegistrationSyN.sh`). Commonly:
- `*_0GenericAffine.mat` - Affine transformation
- `*_1Warp.nii.gz` - Deformable (warp) transformation

## Coordinate Systems

Understanding coordinate systems is crucial for DBS workflows:

### **Frame Space** (Leksell Stereotactic Frame)
- Origin at upper, posterior, right corner of frame
- Units: millimeters
- Defined by ring and arc angles

### **Postop CT Space**
- Physical coordinates in postoperative CT image
- RAS convention (Right-Anterior-Superior)
- Units: millimeters
- Contains visible electrodes but may have metal artifacts

### **Preop CT Space**
- Physical coordinates in preoperative CT image
- RAS convention
- Units: millimeters
- Better image quality, no metal artifacts

### **T1W Space**
- Physical coordinates in T1-weighted MRI
- RAS convention
- Units: millimeters
- Best anatomical reference

### **Transformation Chains**

**Surgical Planning:**
```
Frame Space → [Frame Reg] → CT Space → [ANTs] → T1W Space
```

**Electrode Reconstruction:**
```
Postop CT → [ANTs] → Preop CT → [ANTs] → T1W Space
```

## Common Workflows

### Workflow 1: Surgical Planning → T1W
```bash
# Transform planned targets from frame to T1W
python transform_surgical_csv.py \
    surgical_data.csv \
    frame_registration.json \
    ct_to_t1w_0GenericAffine.mat \
    surgical_data_t1w.csv
```

### Workflow 2: Electrode Localization → T1W
```bash
# Transform detected electrodes from postop CT to T1W (via preop CT)
python transform_electrode_json.py \
    electrode_reconstruction.json \
    postop_to_preop_0GenericAffine.mat \
    preop_to_t1w_0GenericAffine.mat \
    electrode_reconstruction_t1w.json
```

### Workflow 3: Compare Plan vs. Actual
After running both workflows above, you can compare:
- `surgical_data_t1w.csv` - Planned target locations in T1W
- `electrode_reconstruction_t1w.json` - Actual electrode locations in T1W

## API Reference

### `transform_surgical_csv()`
```python
from dbstoolbox.utils.transform_coordinates import transform_surgical_csv

transformed_data = transform_surgical_csv(
    csv_data=csv_data,           # List[Dict] from CSV
    transform_files=[...],        # List[Path] to transform files
    invert_flags=[False, True],  # List[bool] for each transform
    transform_types=['frame_registration', 'ants']  # List[str]
)
```

### `transform_pypacer_reconstruction()`
```python
from dbstoolbox.utils.transform_coordinates import transform_pypacer_reconstruction

transformed_data, num_contacts = transform_pypacer_reconstruction(
    reconstruction_data=data,              # Dict from PyPaCER JSON
    transform_files=[postop_to_preop,      # List[Path] to transform files
                     preop_to_t1w],
    invert_flags=[True, True],             # List[bool] for each transform
    transform_types=['ants', 'ants']       # List[str]
)
```

## Notes

- **Transform Order Matters**: Transforms are applied in the order provided
- **Coordinate Convention**: All coordinates use RAS (Right-Anterior-Superior)
- **Original Data**: Original coordinates are always preserved in output

## Troubleshooting

**ImportError: antspyx not found**
```bash
uv sync  # Installs all dependencies including antspyx
```

**Transform file not found**
- Verify file paths are correct
- Use absolute paths or relative to script directory

**Invalid transform error**
- Ensure transform files are in correct format (.json for frame reg, .mat or .nii.gz for ANTs)
- Check transform files were generated correctly
